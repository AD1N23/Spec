## Задание 1 — Диаграмма деятельности: подготовка к защите ВКР

**Папка:** [`Задание 1/`](Задание%201/)

Создана диаграмма деятельности UML, отображающая полный процесс подготовки студента к защите выпускной квалификационной работы.

**Что отображено:**
- Параллельные процессы: написание текста ВКР, разработка практической части, подготовка презентации
- Итерационный процесс проверки текста научным руководителем с циклом доработки
- Прохождение нормоконтроля
- Получение отзыва руководителя и рецензии
- Финальная защита перед комиссией и ветка повторной защиты

## Задание 2 — Набор UML-диаграмм: Telegram-бот учёта питания

**Папка:** [`Задание 2/`](Задание%202/)

**Предметная область:** Telegram-бот для ведения персонального дневника питания. Пользователь вводит название продукта и вес, бот находит его в базе (~75 наименований), рассчитывает КБЖУ для порции и сохраняет запись в PostgreSQL. Реализован на Go.

### Диаграммы

| Файл | Тип | Сущность | Описание |
|---|---|---|---|
| [`use_case.puml`](Задание%202/use_case.puml) | Use Case | Статическая | Варианты использования: текущий бот и планируемые функции MiniApp |
| [`classes.puml`](Задание%202/classes.puml) | Classes | Статическая | Архитектура приложения: модели, интерфейсы, сервисы, репозитории, хендлеры |
| [`sequence.puml`](Задание%202/sequence.puml) | Sequence | Динамическая | Три ключевых сценария: добавление еды, выбор из совпадений, просмотр и редактирование отчёта |
| [`state.puml`](Задание%202/state.puml) | State | Динамическая | Состояния сессии пользователя: ввод еды, выбор продукта, отчёт, редактирование |
| [`activity.puml`](Задание%202/activity.puml) | Activity | Динамическая | Алгоритм обработки каждого входящего Update от Telegram API |
